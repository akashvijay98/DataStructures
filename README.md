\## 🌟 Problems Solved Using Multiple Intersecting Patterns

Certain complex problems require combining two or more distinct patterns or data structures to achieve optimal constraints. Here are the highlights from the repository:

\* \*\*\`shortestBridge.java\`\*\*: Uses \*\*DFS\*\* to locate and group the first island's coordinates, followed immediately by \*\*BFS\*\* starting from those coordinates to find the shortest path (minimum flips) to the second island.

\* \*\*\`wordBreak2\_memoization.java\`\*\*: Beautifully combines \*\*Backtracking\*\* to construct every valid sentence, while using \*\*Dynamic Programming (Memoization)\*\* to cache previously computed valid substrings to prevent Time Limit Exceeded (TLE).

\* \*\*\`findMedianFromDataStream.java\`\*\*: Integrates \*\*Two Priority Queues\*\* (a Max-Heap for the lower half of numbers and a Min-Heap for the upper half) to stream elements while finding the median in $O(1)$ time.

\* \*\*\`LRUCache\`\*\*: A classic system design problem that merges a \*\*Hash Map\*\* (for $O(1)$ lookups) and a \*\*Doubly Linked List\*\* (for $O(1)$ evictions and updates) to manage cache state.

\* \*\*\`rottingOranges.java\`\*\*: Models a 2D \*\*Matrix\*\* problem but strictly utilizes \*\*Multi-source BFS\*\*, treating all initially rotten oranges as the layer-0 nodes in a Graph.

\* \*\*\`MinimumCostToReachDestinationInTime.java\`\*\*: An advanced Graph problem blending \*\*Dijkstra's Algorithm\*\* (for pathfinding via Priority Queue) combined heavily with \*\*Dynamic Programming\*\* state arrays to track minimal costs under strict time constraints.

\* \*\*\`slidingWindowMaximum.java\`\*\*: Integrates a \*\*Sliding Window\*\* iterating protocol with a \*\*Monotonic Deque\*\* to ensure the maximum element of a window is always retrieved in $O(1)$ amortized time.

\* \*\*\`WordSquare2.java\`\*\*: Uses a \*\*Trie\*\* to rapidly fetch valid prefix words and embeds that logic deep within a \*\*Backtracking\*\* algorithm to construct the square row by row.

\* \*\*\`findKClosestElements.java\`\*\*: Often solved by utilizing \*\*Binary Search\*\* to hone in on the exact insertion index, followed seamlessly by a \*\*Two Pointers\*\* expansion window to extract the \`K\` closest elements.

\* \*\*\`ShortestPathVisitingAllNodes.java\`\*\*: An advanced traversal that maps state via \*\*BFS\*\*, but uses \*\*Bitmasking\*\* to keep track of the uniquely visited path states to avoid infinite cycles.
